**Зачем это было сделано? Чему научился в процессе выполнения задания?**
Потому что инструменты сборки это неотъемлемая часть приложений. Вот что я узнала:

Build Tool - инструмент, не зависимый от среды разработки который компилирует фацлы проекта

Continuous Integration - процесс регулярной сборки проекта на удаленных серверах
Описывается в одном xml файле
Есть зависимости

CI - Для обнаружения ошибок в репозитории

У Maven основная цель - сделать стандартизированную структуру проекта

Gradle - делает настройку билда как можно более flexible и что бы легко можно было кастомизироватт

Инкрементная сборка - в первый раз билдит весь проект, далее - только изменения

Мультимодульный проект - проект из нескольких модулей и модули билдятся отдельно друг от друга. В одном репозитории - несколько сборок
