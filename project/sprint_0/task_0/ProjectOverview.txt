Описание проекта и технологий.

В этом документе мы постараемся кратко описать технологии
применяемые в разработке данного проекта. Мы понимаем, что
для вас этот проект на первый взгляд может показаться сложным
и не понятным. Мы помним это чувство! Когда приходишь на свою
первую работу и тебе говорят: "Вот линк, скачай и установи
проект на свой компьютер!" Вы скачиваете и открываете проект,
а там 100500 файлов. Пора привыкать, ведь так выглядят большинство
реальных проектов.

Далее мы постараемся вкратце описать из чего состоит наш проект.

Начнём с самого простого. Для разработки нашего проекта мы будем
использовать VCS (Version Control System) систему контроля версий.
Все реальные приложения разрабатываются с использованием той или
иной VCS. Для нашего проекта мы будем использовать Git.

Репозиторий нашего проекта расположен на сайте GitHub. Это открытый
репозиторий, права на запись в этот репозиторий получат все участники
проекта. Именно поэтому для участия в проекте вам понадобится GitHub
аккаунт.

В корне проекта лежит файл .gitignore. В этом файле указаны правила,
которые запрещают комитить определённые файлы и папки в Git. Это
сделано для того, чтобы специфические данные участников проекта не
попадали в общий репозиторий.

В проекте для сборки проекта и управления зависимостями применяется
Build Tool Gradle. В корне проекта находится папка gradle в которой
расположены файлы: gradle-wrapper.jar и gradle-wrapper.properties.
gradle-wrapper.jar - файл для запуска Gradle.
gradle-wrapper.properties - файл в котором прописана версия Gradle
используемая в проекте.

В корне проекта лежит файл build.gradle. Это главный файл конфигурации
Build Tool Gradle для нашего проекта. Загляните в него. В нём прописаны
версия JDK, указывается репозиторий от куда наш проект будет брать
сторонние библиотеки.

В корне проекта так же расположен файл settings.gradle. Этот файл
предназначен для регистрации Gradle модулей в проекте. Gradle будет
собирать только те модули, что указаны в этом файле. Для начала
работы над проектом нужно добавить сюда свой модуль.

Код проекта основан на популярном в мире Java фреймворке Spring Boot.
В коде проекта создан один REST контроллер, функционал которого вам
придется разрабатывать.
