Разработка большого приложения на Java
не мыслима без применения инструментов
для сборки проекта. Они называются Build Tools.

В нашем проекте мы будем использовать один
из таких инструментов под названием Gradle.

Gradle это современный гибкий инструмент
предназначенный для сборки и управления
проектами любой сложности (https://gradle.org/).

В задачу Gradle входит предоставление удобных
команд для сборки проекта, а так же работа
с зависимостями (dependencies) и многое другое.

Ваша задача: прослушайте запись лекции про Build Tools,
Maven структуру проекта, Gradle и Continues Integration (CI):
https://www.youtube.com/watch?v=21saM8pvdS4

В этом задании мы познакомимся с основными
файлами Gradle в нашем проекте.

В нашем проекте мы используем Gradle wrapper.
Gradle wrapper это Java библиотека, которая
расположена в папке gradle нашего проекта.
Откройте ее и познакомьтесь с ее содержимым.

В этой папке вы можете найти следующие файлы:
- gradle-wrapper.jar - это сам Gradle wrapper в виде Java библиотеки
- gradle-wrapper.properties - это конфигурация.

Удобство распространения Gradle в виде wrapper
(Java библиотеки) заключается в том, что его не
надо скачивать отдельно. Вы клонируете репозиторий
и сразу получаете Gradle wrapper готовый к работе.

Так как Gradle wrapper это Java библиотека,
то все что нужно для ее работы это
указать в проекте подходящую JDK.

Откройте меню: File -> Project Structure
и убедитесь, что у вас выбрана
SDK 21 (например Amazon Corretto version 21.0.3)
и установлен Language Level: 21 - JavaDoc snippets
или SDK default.

Откройте меню: File -> Settings и в верхнем левом углу
в строке поиска наберите Gradle. Убедитесь, что
Gradle JVM: установлена в Project SDK corretto 21.
В этом же окне установите:
Build and run using: IntelliJ IDEA
Run tests using: IntelliJ IDEA

После установки и конфигурации JDK,
откройте справа закладку Gradle и в
верхней панели выберите иконку с двумя
стрелочками Reload All Gradle Projects.

Если вы все сделали правильно, то в этот
момент запустится Gradle wrapper, он скачает
все зависимости (dependencies) прописанные
в нашем проекте и подключит их к нашему проекту.

PS: не забудьте по окончанию выполнения шага создать и заполнить
файл /documentation/Task_x_description.txt. Формат заполнения этого
файла описан в task_1 в файле Learning_reflection.txt.
